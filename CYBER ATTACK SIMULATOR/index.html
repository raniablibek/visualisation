üìñ Application Guide

üéØ DoS (Denial of Service)

A DoS attack overwhelms a single server with requests from one source.

  • Intensity: Controls the rate of attack requests
  • Capacity: Server's ability to handle requests
  • Defense: Percentage of requests blocked by security
  • Warning: At 70% load, a warning appears indicating server stress

üåê DDoS (Distributed Denial of Service)

A DDoS attack uses multiple compromised systems (botnet) to flood a target.

  • Bots: Number of compromised machines in the botnet
  • Per bot: Requests per second from each bot
  • Capacity: Maximum traffic the server can handle
  • Defense: Percentage of malicious traffic blocked

üìß Phishing

Phishing simulates social engineering attacks via email.

  • Email Received: Simulated phishing emails sent
  • Clicks: Users who clicked malicious links
  • Alerts: Security detections triggered
